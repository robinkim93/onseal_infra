# chunking 직접 구현 · embedding 에어갭 모델 로딩 · 파이썬 문법(TS 대조)

- **날짜:** 2026-06-16
- **단계:** M1 (에어갭 RAG MVP)
- **모드:** tutor(파이썬 문법·에어갭·계약 개념 주력) · reviewer→delegate(chunk_text·embed 완성 예시 제공) · 메타(사용자 프로파일 보정)

## 한 줄 흐름

학습 재시작 후 RAG 데이터 경로(chunking→embedding) 구현에 착수. 도중 "파이썬 기초부터 설명"으로
튜터링을 잘못 보정했다가 **사용자 피드백(TS 백엔드 3년 — 개념은 능숙, 갭은 파이썬 문법뿐)**으로
교정하고 CLAUDE.md·메모리에 프로파일을 박았다. chunking은 글자수 기준 고정크기+오버랩으로 직접
구현했고, embedding은 **에어갭 절대제약이 코드에서 처음 실제로 걸리는 지점**(Hub ID = 런타임
다운로드)을 짚어 로컬 경로 로딩 + 1회 로드(lazy singleton) 패턴으로 닫았다.

## 헷갈림 → 교정 궤적  ★핵심

| 헷갈린/몰랐던 것 | 내가 내놓은 답·추론 | 정확한 개념 |
|---|---|---|
| 바디가 비었는데 출력 모양을 어떻게 아나 | "내부에 코드가 없는데 어떤 모양인지 모르지" | **반은 정답.** 모양(타입/계약 `-> list[str]`)은 코드 없이도 **시그니처가 약속**. 실제 내용물(값)만 코드가 정함. 자판기 라벨(약속) vs 내부 기계(값). TS의 타입 시그니처 vs 구현 감각 그대로 |
| 청킹 = 단어 분리? | "안녕하세요 여러분 → ['안녕하세요','여러분'] 겹침 있어야?" | **청킹 ≠ 토크나이징.** 우리 건 글자수 기준 슬라이싱. `"안녕하세요 여러분"`(8자<500)은 **1개 조각**, 겹침 없음. 겹침은 조각이 2개↑ 생길 때만 의미 |
| range step 값 | "450씩?" | 정답. `step = size - overlap = 450`. 그래야 직전 조각 끝 50글자가 다음 조각 앞에 재등장 |
| 1000자 → 몇 조각 | "0~500, 450~950, 900~1000 = 3개" | 정답. `range(0,1000,450)` → i=0,450,900 → 슬라이스 3개. 마지막은 `[900:1000]` 100자 |
| 임베딩 모델 로드 위치 | "main.py에서 1회? / rag/__init__.py에서?" | 핵심(함수 밖·1회 재사용) 정확. 위치별 트레이드오프 존재: 모듈 최상단(표준) / `__init__.py`(import=무거운 부작용 숨음) / FastAPI startup(명시적) |
| `SentenceTransformer("BAAI/bge-m3")` 에어갭 문제 | "외부 api 호출인데 에어갭은 외부 통신이 안 됨" | 정확. Hub **ID**(경로 아님)→로컬에 없으면 HuggingFace Hub에서 **런타임 다운로드** 시도→폐쇄망에서 실패 |
| Hub ID 대신 뭘 넘기나 | "파일 경로로?" | 정확. **로컬 경로**로 가리키면 "받기"가 "읽기"로. + `HF_HUB_OFFLINE=1`로 "조용히 네트워크" 대신 "시끄럽게 실패" 안전장치 |
| `global` 빼면? | "함수 내부에서 새로 생성, 외부 model은 계속 None" | 방향 정확. 파이썬 심화: **함수 안에서 대입되는 이름은 함수 전체에서 지역변수 취급** → 윗줄 `if _model is None` 읽기조차 `UnboundLocalError`로 **즉시 터짐**(조용한 재로드 전에 막아줌) |

## 다룬 개념

| 개념 | 한 줄 핵심 |
|---|---|
| 시그니처=계약 vs 바디 | 타입(모양)은 코드 없이도 약속됨, 실제 값만 코드가 정함 (TS 타입 vs 구현) |
| 청킹(글자수 슬라이싱) | size 고정, `text[i:i+size]`, 슬라이싱은 범위 초과해도 안전(있는 데까지) |
| 청킹 ≠ 토크나이징 | 단어 분리 아님. size보다 짧으면 1조각 |
| 오버랩 | 경계 걸친 정보 보존, step=size-overlap, 대가는 중복 저장 |
| 파이썬 문법 (TS 대조) | `len()`(함수), `text[i:j]`(=slice), `range(start,stop,step)`(stop 미포함), `for in`, `[]`/`.append()`, `:`+들여쓰기=블록 |
| 에어갭 모델 로딩 | Hub ID→런타임 다운로드(위반). 로컬 경로 + 빌드(네트워크 OK)/런타임(0) 시점 분리 + `HF_HUB_OFFLINE` |
| 모듈 import 1회 캐시 | 파이썬은 모듈을 프로세스당 1회 import(`sys.modules`) → 모듈 스코프 변수=1회 실행 (TS top-level const와 동일) |
| lazy singleton | `_model=None` + `if _model is None` + `global` → 첫 호출에만 로드, 이후 재사용 |
| `global` 함정 | 함수 내 대입 이름=지역 취급 → `global` 없이 모듈 변수 재대입 시 `UnboundLocalError` |
| `is None` 관용구 | None 비교는 `==` 아닌 `is`(프로세스 단일 객체, 동일성 비교) |
| `normalize_embeddings` | 벡터 길이 1로 정규화. 코사인(`<=>`)엔 순위 영향 없지만 관용적 켜둠(내적 전환 여지) |
| `.tolist()` | `encode()`는 numpy 배열 반환 → 계약(`list[list[float]]`)·DB·JSON 위해 순수 리스트로 |
| 설정 한 곳 변경 | `settings.embedding_model`: dev=Hub ID / 에어갭=로컬 경로. 코드 불변, 설정만 |

## 막힌 곳 → 해결

- **튜터링 오보정(메타):** 증상=함수·리스트 같은 프로그래밍 기초를 처음부터 설명 → 사용자 피드백("TS 3년, 개념 능숙, 파이썬 문법만 갭") → CLAUDE.md "사용자 프로파일(튜터링 보정)" 절 + 메모리 `user-career-goal` 갱신. 이후 파이썬 문법은 TS 대조로만, 기초 설명 금지.
- **부품→조립 막힘:** chunk_text를 문법 부품만으로 조립하려다 "전혀 모르겠다" → 방식 전환: **주석 단 완성 예시를 읽고→머릿속 트레이스→변형**. 1000자→3조각 트레이스로 이해 확인.
